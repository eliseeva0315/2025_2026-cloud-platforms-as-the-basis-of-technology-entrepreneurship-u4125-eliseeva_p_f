# Лабораторная работа №4  
**Разработка инфраструктуры MVP AI-приложения**

## Цель работы  
Создать прототип AI-приложения (чат-бота) с базовой функциональностью: спроектировать инфраструктуру, рассчитать экономическую модель и обосновать выбор облачных ресурсов для трёх стадий жизненного цикла.

---

## Описание приложения  

**Название:** AI чат-бот для генерации поздравлений  

**Тип системы:** веб-приложение / чат-бот  

**Цель:**  
Генерация текстов поздравлений (день рождения, праздники, корпоративные события) на основе пользовательского запроса.

**AI-компонент:**  
Использование модели **Vertex AI (Gemini)** для генерации текста.

# Стадия 1 — MVP  

## Цель  
Проверка идеи с минимальными затратами.

## Стратегия  
Использование serverless-архитектуры и готового AI API.

---

## Инфраструктура (GCP)

| Ресурс | Конфигурация | Обоснование |
|--------|-------------|------------|
| Cloud Run | 1 vCPU, 512 MB | Платим только за запросы |
| Vertex AI (Gemini Flash) | API | Нет необходимости в GPU |
| Cloud SQL | db-f1-micro | Минимальная база |
| Cloud Storage | ~5 GB | Хранение данных |
| Cloud Monitoring | Free tier | Логи |

# Обоснование выбора  

Cloud Run выбран для MVP, так как не требует оплаты за простой и автоматически масштабируется.

Vertex AI позволяет использовать готовую модель без настройки инфраструктуры.

Cloud SQL обеспечивает надежную базу данных без администрирования.

Redis используется для ускорения работы и ограничения запросов.

GKE применяется на этапе Production для обработки высокой нагрузки.

Cloud Armor добавляется для защиты от атак.
